# 45GS02 opcode dataset notes (v0.1)

## Scope
This is an initial machine-readable dataset scaffold covering the subset currently implemented in `src/m45gs02/`.
It is intentionally incomplete and must be expanded to full 4510/45GS02 coverage.

## Provenance anchors (mega65 extract)
- Appendix K entry: `45GS02 Microprocessor` at `documentation/_pdf_extract/mega65-book-2.txt:622`
- Appendix L entry: `45GS02 & 6502 Instruction Sets` at `documentation/_pdf_extract/mega65-book-2.txt:656`
- 4510 instruction set heading: `documentation/_pdf_extract/mega65-book-2.txt:24952`
- 4510/45GS02 opcode table heading: `documentation/_pdf_extract/mega65-book-2.txt:25012`
- Compound instruction chapter heading: `documentation/_pdf_extract/mega65-book-2.txt:27744`

These line anchors are generated by `documentation/45gs02/extract_anchor_lines_v0_1.py`
and persisted in `documentation/45gs02/anchor_lines_v0_1.json`.

## Raw opcode-table extraction
- `documentation/45gs02/extract_4510_opcode_rows_v0_1.py` extracts OCR table rows from the
	Appendix L region (`25012..27744`) into
	`documentation/45gs02/opcode_table_4510_45gs02_raw_v0_1.csv`.
- Current extraction result: `224` rows (plus header), including `relfar`, `ibpZQ`, `MAP`, and `EOM` entries.

## Dataset conventions
- `address_mode_key` uses normalized opForge-family naming where possible.
- `prefix_sequence` is one of: `none`, `NOP`, `NEG NEG`, `NEG NEG NOP`.
- `opcode_hex` is the base opcode after prefix lowering.
- Q mnemonics are represented as pseudo rows with explicit prefix requirements.
